# Vigil 0.1.0

First public release of Vigil — a macOS menu-bar app and CLI for keeping a Mac fully awake while the lid is closed. Designed for running long jobs (AI agents, builds, syncs) without leaving the laptop propped open.

## What's in this release

- Reversible `pmset` awake profile (`disablesleep`, `sleep`, `disksleep`, `ttyskeepawake`, `tcpkeepalive`).
- Persistent user LaunchAgent that takes IOKit power assertions only while enabled.
- Lid-state watcher that dims displays + keyboard backlight on close and restores on open.
- One-time scoped sudoers approval so Enable / Disable run without repeated password prompts.
- Privileged-helper version-mismatch detection prompts re-approval after Sparkle updates.
- App Translocation guard refuses to enable from quarantined paths (drag to /Applications first).
- Sparkle 2 auto-update from GitHub Releases (EdDSA-signed appcast).

## Install

1. Download `Vigil-0.1.0.dmg` from this release page.
2. Open the DMG and drag `Vigil.app` to the `Applications` shortcut inside it.
3. Open `Vigil.app` from `/Applications`. The menu-bar icon appears at the right of the status bar.

If you prefer the `.zip` artefact instead: unzip it and **drag `Vigil.app` to `/Applications` before opening it.** Launching from `~/Downloads` triggers macOS App Translocation; Vigil refuses to enable in that state.

## Auto-update

Vigil checks for updates daily via Sparkle 2. Updates are EdDSA-signed and Developer-ID + notarized by Apple. You can disable auto-update in the Sparkle preferences dialog.

## Safety

Closed-lid operation can trap heat. Keep the machine ventilated and prefer AC power. Vigil refuses to enable on battery power unless you click "Enable on Battery".
