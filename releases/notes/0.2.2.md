## Battery floor — both features

Both `vigil lid-awake` and `vigil caffeinate` now support a **battery floor**: while running on battery power, the agent disables the feature once the battery drops to a configured percentage so the Mac can sleep normally instead of running flat.

- **CLI**: `--battery-floor <int>` (range 1–99) on `vigil <feature> on` and `toggle`. Works for both features.
- **Menu app**: new **Disable on low battery** toggle + percentage picker inside each feature card.
- **Defaults**:
  - **Lid-Awake: on at 20%.** The unattended-by-design case — closed lid, user walked away. The safety net is unambiguous.
  - **Caffeinate: off (opt-in).** The user-is-present case — auto-disabling mid-presentation/video would be more surprising than helpful. The toggle is right there if you want it.
- **Trip is one-way**: plugging AC back in does NOT auto-re-enable.
- **Requirements**: Lid-Awake's floor requires `vigil approve-all` so the agent can restore the privileged `pmset` profile non-interactively. Caffeinate has no `pmset` to restore — its trip handler just releases its IOKit assertions and exits.

If Lid-Awake's privileged restore fails at trip time (e.g. approval was revoked mid-session), the agent retries twice on 30-second intervals, then exits. `vigil status` honestly reports the degraded state until you run `vigil lid-awake off` to restore power settings interactively, and the menu app's notification says `Lid-Awake ended — power settings could not be restored. Run \`vigil doctor\`.` instead of pretending the safety feature worked.

## Diagnostics

- `vigil status` (and the JSON status payload) surface the configured battery floor on the active session, plus the most recent end reason (`user-disabled` / `timer-expired` / `interrupted` / `battery-threshold`) per feature.
- Reason-aware notification copy: `Lid-Awake disabled — battery low.` / `Caffeinate disabled — battery low.` instead of the generic ending notification when the battery floor trips cleanly.

## Compatibility

- Additive only — no schema bump, no IPC contract bump, no breaking CLI / JSON changes.
- Sessions persisted by 0.2.1 decode unchanged.
- The `EndReason` decoder is now forward-compatible: future enum cases (0.2.3+) will land as `unknown` rather than silently dropping `sessionEnded` events from the stats log.
